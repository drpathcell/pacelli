# Pacelli Full Project Audit — 4 April 2026

**Scope:** Security, code quality, architecture, test coverage, CI/CD, dependency health, build config, Cloud Functions.
**Codebase:** 118 Dart source files, 2 test files, 70 Cloud Functions, 30 direct dependencies.

---

## Executive Summary

Pacelli has **strong security foundations** — AES-256-CBC with HKDF, comprehensive Firestore rules, and a well-ordered burn flow. The architecture is clean with consistent Riverpod patterns and proper dual-backend abstraction.

The critical gaps were in **engineering infrastructure**: zero CI/CD, 1.7% test coverage, debug signing on release builds, and no code obfuscation. Most of these have now been addressed.

## Remaining Open Items

### 9. Cloud Functions Logging (MEDIUM — needs deploy)
70 functions with minimal logging. Add structured logging with request ID, operation name, user context before next deploy.

### 12. Large Screen Components (MEDIUM — refactor)
`ai_assistant_screen.dart` (956 lines) should be split into 3–4 widgets. 8 screens exceed 500 lines. No functional risk, but harder to test and maintain.

### 13. Route Auth Guards (LOW-MEDIUM)
GoRouter needs a `redirect:` callback checking `FirebaseAuth.instance.currentUser`. Splash screen handles initial routing, but deep navigation can bypass it.

## What's Working Well

- **Encryption architecture** — AES-256-CBC with HKDF, fresh IVs per message, secure key lifecycle, cross-language compatibility tested
- **Firestore security rules** — comprehensive household_id denormalization, deterministic member doc IDs, default-deny, no wildcard allows
- **Burn data flow** — correct batch ordering (own docs last), retry with throw on failure, `terminate()` + `clearPersistence()`, verification query post-burn
- **Dual backend abstraction** — clean DataRepository interface, both implementations complete (65 methods each)
- **Provider structure** — no circular deps, all FutureProviders use `.autoDispose`, clean DAG
- **Localisation** — 1,272 keys, all 3 locales (EN/ES/IT) in sync, zero missing keys
- **Notification privacy** — generic titles, no sensitive data on lock screen
- **Cloud Functions auth** — ID token verification + household membership + rate limiting on all 70 endpoints
